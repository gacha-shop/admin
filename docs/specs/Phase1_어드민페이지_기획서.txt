어드민 관리자 페이지
Phase 1 - 기본 데이터 관리 시스템
작성일: 2025년 10월

1. 개요
1.1 목적
Phase 1 어드민 페이지는 가챠샵 위치 데이터를 효율적으로 관리하기 위한 내부 도구입니다. 초기 데이터 수집 및 검증을 위한 필수 기능에 집중하되, 향후 Phase 2, 3의 확장 기능을 수용할 수 있는 구조로 설계합니다.
1.2 Phase별 기능 범위


2. Phase 1 핵심 기능
2.1 로그인 및 인증
기능 요구사항
간단한 ID/Password 로그인 (초기에는 1-2명 관리자만 사용)
세션 유지 (로그인 상태 24시간)
비밀번호 변경 기능
향후 확장 고려사항
Phase 2: 역할 기반 권한 관리 (관리자, 에디터, 뷰어)
Phase 3: 매장 운영자 계정 분리

2.2 대시보드
기능 요구사항
한눈에 서비스 현황을 파악할 수 있는 요약 정보 제공
통계 카드 (4개)
- 총 매장 수
- 검증 완료 매장 수
- 미검증 매장 수
- 최근 7일 추가된 매장 수
지역별 분포 차트
- 시/도별 매장 수 막대 그래프
최근 활동 로그
- 최근 추가/수정된 매장 5개 표시 (시간순)
빠른 액션 버튼
- 새 매장 추가
- 미검증 매장 목록 보기
화면 구성
[상단] 통계 카드 4개 (가로 배치)
[중간] 지역별 분포 차트 + 최근 활동 로그
[하단] 빠른 액션 버튼

2.3 매장 관리 (핵심 기능)
2.3.1 매장 목록 조회
기능 요구사항
테이블 형태로 매장 목록 표시
표시 컬럼: ID, 매장명, 주소, 매장 타입, 검증 상태, 등록일, 액션
필터링
- 매장 타입 (전체 / 가챠샵 / 피규어샵 / 둘 다)
- 검증 상태 (전체 / 검증 완료 / 미검증)
- 지역 (시/도 드롭다운 → 시/군/구 드롭다운)
검색
- 매장명 또는 주소로 검색
정렬
- 등록일, 매장명, 검증 상태 기준 정렬 (오름차순/내림차순)
페이징
- 페이지당 20개 / 50개 / 100개 선택 가능
일괄 작업
- 체크박스 선택 → 일괄 검증 완료 처리
- 체크박스 선택 → 일괄 삭제
CSV/Excel 내보내기
- 현재 필터링된 목록을 CSV 파일로 다운로드

2.3.2 매장 추가
기능 요구사항
입력 폼 구성


주소 입력 프로세스
1. 주소 검색 버튼 클릭 → 카카오 주소 팝업
2. 주소 선택 시 자동으로 주소, 위도, 경도, 지역 정보 입력
3. 상세 주소는 수동 입력
4. 지도 미리보기로 위치 확인 가능
유효성 검증
- 필수 항목 미입력 시 에러 메시지
- 중복 매장 체크 (같은 주소 존재 시 경고)
- 위도/경도 유효 범위 체크

2.3.3 매장 수정
기능 요구사항
매장 목록에서 수정 버튼 클릭 → 수정 폼으로 이동
추가 폼과 동일한 구성 (기존 데이터 자동 입력)
수정 이력 자동 기록 (수정 시간, 수정한 관리자)
검증 상태 토글 (미검증 ↔ 검증 완료)

2.3.4 매장 삭제
기능 요구사항
소프트 삭제 방식 (실제 DB 삭제 X, is_deleted 플래그)
삭제 확인 팝업 (실수 방지)
삭제된 매장 복구 기능 (삭제된 매장 목록 별도 페이지)

2.3.5 매장 상세 보기
기능 요구사항
매장명 클릭 시 상세 페이지로 이동
모든 매장 정보 표시 (읽기 전용)
구글 지도 임베드 (위치 표시)
수정 이력 로그 표시 (생성일, 최종 수정일, 검증일)
빠른 액션 버튼 (수정, 삭제, 검증 토글)

2.4 일괄 데이터 가져오기
기능 요구사항
CSV 파일 업로드
- 템플릿 CSV 다운로드 기능 제공
- 필수 컬럼: 매장명, 매장타입, 주소, 위도, 경도, 데이터출처
데이터 검증
- 업로드 후 미리보기 (에러 행 하이라이트)
- 중복 체크 (주소 기준)
- 유효성 검증 (위도/경도 범위, 필수값)
일괄 등록
- 검증 통과한 데이터만 일괄 등록
- 등록 결과 리포트 (성공/실패 건수)

2.5 설정
기능 요구사항
비밀번호 변경
시스템 정보 (버전, 최종 업데이트 날짜)

3. 화면 구성 및 UI/UX
3.1 레이아웃 구조
기본 레이아웃
[좌측] 사이드바 네비게이션
- 대시보드
- 매장 관리
- 일괄 가져오기
- 설정
- 로그아웃
[상단] 헤더 (페이지 제목, 사용자 정보)
[중앙] 메인 컨텐츠 영역

3.2 디자인 가이드
색상 팔레트
- Primary: #0066CC (파랑)
- Success: #00AA00 (초록)
- Warning: #FF9900 (주황)
- Danger: #CC0000 (빨강)
- Neutral: #666666 (회색)
타이포그래피
- 본문: Noto Sans KR / Arial
- 숫자: Roboto
버튼 스타일
- Primary 버튼: 주요 액션 (저장, 등록)
- Secondary 버튼: 보조 액션 (취소)
- Danger 버튼: 삭제 액션

6. 향후 확장 설계
6.1 Phase 2 확장 포인트
Phase 2에서 추가될 기능들을 위한 준비 사항
사용자 제보 시스템
- shop_reports 테이블 활용
- 어드민에서 제보 검토 및 승인/거절 페이지 추가
이미지 관리
- shop_images 테이블 활용
- 매장 상세 페이지에 이미지 업로드 기능
- AWS S3 또는 Cloudinary 연동
역할 기반 권한
- users 테이블에 role 컬럼 추가 (admin, editor, viewer)
- 권한별 접근 제어 미들웨어 구현
고급 통계
- 시계열 차트 (매장 증가 추이)
- 데이터 출처별 통계

6.2 Phase 3 확장 포인트
매장 운영자 계정
- 매장별 owner 계정 생성

품목 관리
- shop_products 테이블 활용
- 매장별 품목 CRUD 페이지
리뷰 관리
- shop_reviews 테이블 활용
- 부적절한 리뷰 신고/삭제 기능
